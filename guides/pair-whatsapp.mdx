---
title: "Pair WhatsApp"
description: "Start pairing, render the QR code, and monitor connection status."
---

Pairing connects one WhatsApp account to one WhatsApp Use connection.

## Recommended flow

<Steps>
  <Step title="Create or select a connection">
    In the dashboard, create a WhatsApp connection for the organization. If you use the API, make sure your API key is allowed to use the target connection.
  </Step>
  <Step title="Request a pairing QR">
    Call `POST /v1/whatsapp/pair` with an API key that has `whatsapp:write`.
  </Step>
  <Step title="Render the QR code">
    Poll `GET /v1/whatsapp/pair/status` and render the returned `qr` value as a QR code.
  </Step>
  <Step title="Scan with WhatsApp">
    On the phone, open WhatsApp linked devices and scan the QR code.
  </Step>
  <Step title="Confirm status">
    Call `GET /v1/whatsapp/status` until `paired` is `true`.
  </Step>
</Steps>

<CodeGroup dropdown>

```bash cURL
curl -X POST \
  -H "Authorization: Bearer $WHATSAPP_USE_API_KEY" \
  "$WHATSAPP_USE_BASE_URL/v1/whatsapp/pair"
```

```python Python
import os
import requests

res = requests.post(
    f"{os.environ['WHATSAPP_USE_BASE_URL']}/v1/whatsapp/pair",
    headers={"Authorization": f"Bearer {os.environ['WHATSAPP_USE_API_KEY']}"},
)
res.raise_for_status()
print(res.json())
```

```javascript JavaScript
const res = await fetch(`${process.env.WHATSAPP_USE_BASE_URL}/v1/whatsapp/pair`, {
  method: "POST",
  headers: { Authorization: `Bearer ${process.env.WHATSAPP_USE_API_KEY}` },
});

console.log(await res.json());
```

```go Go
package main

import (
	"fmt"
	"io"
	"net/http"
	"os"
)

func main() {
	req, _ := http.NewRequest("POST", os.Getenv("WHATSAPP_USE_BASE_URL")+"/v1/whatsapp/pair", nil)
	req.Header.Set("Authorization", "Bearer "+os.Getenv("WHATSAPP_USE_API_KEY"))

	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		panic(err)
	}
	defer resp.Body.Close()
	out, _ := io.ReadAll(resp.Body)
	fmt.Println(string(out))
}
```

## Pairing state

The pairing status response contains:

| Field | Meaning |
| --- | --- |
| `status` | Pairing state such as `qr`, `paired`, or `failed`. |
| `qr` | QR payload to render when `status` is `qr`. |
| `error` | Error text when pairing fails. |

<Info>
  MCP OAuth does not show the WhatsApp QR code. Pair from the platform dashboard or your own product UI, then authorize MCP clients after the connection exists.
</Info>
