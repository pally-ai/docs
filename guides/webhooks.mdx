Webhook endpoints let your application receive events from WhatsApp Use.

## Create an endpoint

Use an API key with `webhooks:manage`.

<CodeGroup dropdown>

```bash cURL
curl -X POST \
  -H "Authorization: Bearer $WHATSAPP_USE_API_KEY" \
  -H "Content-Type: application/json" \
  "$WHATSAPP_USE_BASE_URL/v1/webhooks/endpoints" \
  -d '{
    "url": "https://example.com/webhooks/whatsapp-use",
    "description": "Production webhook",
    "event_types": ["*"],
    "enabled": true
  }'
```

```javascript JavaScript
const res = await fetch(`${process.env.WHATSAPP_USE_BASE_URL}/v1/webhooks/endpoints`, {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.WHATSAPP_USE_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    url: "https://example.com/webhooks/whatsapp-use",
    description: "Production webhook",
    event_types: ["*"],
    enabled: true,
  }),
});

console.log(await res.json());
```

```python Python
import os
import requests

res = requests.post(
    f"{os.environ['WHATSAPP_USE_BASE_URL']}/v1/webhooks/endpoints",
    headers={"Authorization": f"Bearer {os.environ['WHATSAPP_USE_API_KEY']}"},
    json={
        "url": "https://example.com/webhooks/whatsapp-use",
        "description": "Production webhook",
        "event_types": ["*"],
        "enabled": True,
    },
)
res.raise_for_status()
print(res.json())
```

```go Go
package main

import (
	"bytes"
	"fmt"
	"io"
	"net/http"
	"os"
)

func main() {
	body := []byte(`{"url":"https://example.com/webhooks/whatsapp-use","description":"Production webhook","event_types":["*"],"enabled":true}`)
	req, _ := http.NewRequest("POST", os.Getenv("WHATSAPP_USE_BASE_URL")+"/v1/webhooks/endpoints", bytes.NewReader(body))
	req.Header.Set("Authorization", "Bearer "+os.Getenv("WHATSAPP_USE_API_KEY"))
	req.Header.Set("Content-Type", "application/json")

	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		panic(err)
	}
	defer resp.Body.Close()
	out, _ := io.ReadAll(resp.Body)
	fmt.Println(string(out))
}
```

</CodeGroup>

The response includes a `secret`. Store it when you create the endpoint. Use it to verify webhook signatures.

## Delivery format

Webhook requests use JSON:

```json
{
  "id": "evt_20260606173000.000000000",
  "type": "whatsapp.connection.status_changed",
  "source": "whatsapp",
  "user_id": "org_...",
  "api_key_id": "key_...",
  "payload": {
    "status": "paired"
  },
  "occurred_at": "2026-06-06T17:30:00Z"
}
```

## Signature headers

Each delivery includes:

| Header | Description |
| --- | --- |
| `X-Whatsapp-Use-Event-Id` | Event ID. |
| `X-Whatsapp-Use-Event-Type` | Event type. |
| `X-Whatsapp-Use-Timestamp` | Unix timestamp used for signing. |
| `X-Whatsapp-Use-Signature` | Signature in the form `t=<timestamp>,v1=<signature>`. |
