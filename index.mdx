WhatsApp Use gives developers a managed way to connect a WhatsApp account to software. You create an organization, create a WhatsApp connection, pair it with a QR code, and then use REST API calls or a remote MCP server to read and act on WhatsApp data.

The platform has two public interfaces:

<CardGroup cols={2}>
  <Card title="REST API" icon="braces" href="/quickstart">
    Use API keys to pair WhatsApp, send messages, read conversations, fetch media on demand, and manage webhooks.
  </Card>
  <Card title="Remote MCP" icon="bot" href="/mcp/overview">
    Let AI clients use WhatsApp tools through OAuth and a runtime-bound MCP grant.
  </Card>
</CardGroup>

## How it works

<Steps>
  <Step title="Create an organization">
    Sign in to the platform and choose or create an organization. API keys, WhatsApp connections, MCP grants, usage, and webhooks belong to the organization.
  </Step>
  <Step title="Create a WhatsApp connection">
    A connection represents one paired WhatsApp account. An organization can have more than one connection.
  </Step>
  <Step title="Pair WhatsApp">
    Start pairing from the dashboard or API, then scan the QR code with the WhatsApp mobile app.
  </Step>
  <Step title="Use REST or MCP">
    REST clients use API keys. MCP clients use OAuth and receive access to an already-owned connection.
  </Step>
</Steps>

<Info>
  WhatsApp device pairing is separate from MCP OAuth. OAuth authorizes an AI client to use your existing connection. Pairing authorizes the backend runtime to connect to WhatsApp.
</Info>

## Products

WhatsApp Use currently exposes the WhatsApp API and WhatsApp MCP server. The agent-in-a-box runtime and HelixDB memory system are not part of the public product surface yet.
